## 🪐 What is MANTIS?

**MANTIS OS** (Machine-Assisted Neural Technical Intelligence System) is a **next-generation AI diagnostic platform** that transforms how companies deliver hardware support.

Traditional support is slow, expensive, and inconsistent. MANTIS replaces that with an intelligent, voice-enabled, 3D-visualized system that:

- 📄 **Ingests** product manuals, service documents, and technical specs
- 🧠 **Learns** the nuances of each device using vector-based RAG retrieval
- 🗣️ **Speaks** its diagnostic analysis in real-time using AI-native voice output
- 📷 **Scans** images of broken hardware via Computer Vision overlays
- 🚁 **Dispatches** field technicians automatically upon high-confidence diagnosis

MANTIS doesn't just answer questions — it conducts full diagnostic sessions, builds live neural confidence graphs, and bridges the gap between a user with a broken device and the nearest qualified technician.

## ⚡ Quick Start

### Prerequisites

- **Node.js** ≥ 18.x
- **npm** ≥ 9.x

### 1. Clone & Install

```bash
git clone https://github.com/devillikevd/mantis1.git
cd mantis1
npm install --legacy-peer-deps
```

### 2. Configure Environment

```bash
cp .env.example .env
```

Edit `.env`:

```env
# For local development (SQLite — no cloud needed)
DATABASE_URL="file:./dev.db"

# Auth secret (any random string)
AUTH_SECRET="your-secret-here"
NEXTAUTH_URL="http://localhost:3000"
```

### 3. Initialize Database

```bash
# Generate Prisma client
npx prisma generate

# Create tables
npx prisma db push

# Seed with demo data
npm run db:seed
```

### 4. Start Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and log in with `demo@company.com` / `demo123`.

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/devillikevd/mantis1)

1. Go to [neon.tech](https://neon.tech) → Create free account
2. Create new project → Copy the **PostgreSQL connection string**

#### Step 2: Update Schema

Change `prisma/schema.prisma` provider to `postgresql`:

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

#### Step 3: Deploy on Vercel

1. Push code to GitHub
2. Go to [vercel.com/new](https://vercel.com/new) → Import your repo
3. Add **Environment Variables**:

| Variable | Value |
| :------- | :---- |
| `DATABASE_URL` | Your Neon PostgreSQL URL |
| `AUTH_SECRET` | Random 32+ char string |
| `NEXTAUTH_URL` | `https://your-app.vercel.app` |

4. Click **Deploy** — Vercel auto-runs `prisma generate && next build`

#### Step 4: Seed Production Database

```bash
# Set your production DATABASE_URL in .env temporarily
npm run db:seed
```

## 🗃️ Database Schema

```prisma
model User {
  id           String    @id @default(cuid())
  email        String    @unique
  name         String?
  passwordHash String?
  role         String    @default("USER")   // "USER" | "COMPANY"
  avatar       String?
  createdAt    DateTime  @default(now())
  updatedAt    DateTime  @updatedAt
  companies    Company[]
}

model Company {
  id         String    @id @default(cuid())
  name       String
  slug       String    @unique             // URL-safe name
  gstNumber  String?                       // Indian GST number
  website    String?
  userId     String                        // Owner reference
  products   Product[]
}

model Product {
  id          String   @id @default(cuid())
  name        String
  slug        String   @unique
  description String
  image       String?                      // Unsplash URL
  price       Float
  status      String   @default("ACTIVE")  // "ACTIVE" | "INACTIVE"
  companyId   String
}
```

## 🧬 AI & Diagnostic Engine

### How the RAG Pipeline Works

```
User Message
     │
     ▼
Extract Intent + Product Context
     │
     ▼
Vector Search (Moss / FAISS)
     │
  ┌──┴───────────────────────────┐
  │  Top-K Relevant Chunks       │
  │  from Product Manuals / PDFs │
  └──────────────────────────────┘
     │
     ▼
Construct Augmented Prompt
     │
     ▼
LLM (GPT-4 / Gemini Pro)
     │
     ▼
Stream Response + Update Confidence
```

### Diagnostic Phases

| Phase | Description | Confidence Range |
| :---- | :---------- | :--------------- |
| `SYMPTOM_CAPTURE` | Gathering fault description | 0–30% |
| `ANALYSIS` | Cross-referencing manuals | 30–60% |
| `FAULT_ISOLATION` | Pinpointing root cause | 60–85% |
| `RESOLUTION` | Providing fix steps | 85–100% |

### Confidence → UI State

```
0–30%   →  Blue gauge, normal chat input
30–60%  →  Yellow gauge, quick replies appear
60–85%  →  Orange gauge, "Root Cause Confirmed" overlay
85–100% →  Green gauge, DISPATCH TECHNICIAN button
```
